Executive Summary

Overall, the workspace contains mostly small models with generally sound star schemas where present, but it has material refresh and governance risks driven by (a) local file-path datasources that will not refresh in the service, (b) inconsistent security posture (RLS absent in most models), and (c) Power Query folding risks in at least two import models. Key Findings (Evidence-Based)

1) Refresh reliability risk: local file paths

  • S&P Company Report uses local paths (e.g., C:\Users\...\sp500_*.csv) flagged as non-portable → service refresh will fail without a gateway and stable path mapping.
  • TableauWorkbooks uses local Excel (C:\Users\...\Working Sessions List.xlsx) flagged as non-portable → same risk.

2) Query folding risk in import models

  • 11-Solution-Sales Analysis / Sales: a Table.AddColumn occurs before an explicit filter step; analysis notes no Table.SelectRows filter detected in the query, increasing chance of client-side processing and larger data movement.
  • NYC Taxi - Not Folded: multiple transforms (DuplicateColumn, SplitColumn, ReorderColumns) occur before filtering; flagged as likely folding breakers.

3) Governance: RLS absent in most models

  • Only 11-Solution-Sales Analysis defines RLS (role “Salespeople”) filtering Salesperson (Performance) by [UPN] = USERPRINCIPALNAME().
  • All other models indicate no roles defined.

4) Relationship risk: inactive & bidirectional

  • 11-Solution-Sales Analysis has 1 inactive relationship (Sales.EmployeeKey → Salesperson (Performance).EmployeeKey) and 1 bidirectional cross-filter.
  • TableauWorkbooks has 1 bidirectional cross-filter.

5) Consistency: naming with spaces (programmatic friction)

  • Multiple models include object names with spaces (e.g., “Salesperson (Performance)”, “Workbook List”, measures like “average revenue per trip”). This increases friction for tooling, DAX conventions, and automation.

6) Reuse opportunity: shared Fabric SQL DB used by 3 models

  • Datasource inventory shows a single Sql.Database endpoint consumed by NYC Taxi - DQ, NYC Taxi - Folded, and NYC Taxi - Not Folded → strong candidate for centralized ingestion/curation.

Eliminate or strictly justify bidirectional cross-filter relationships

Applies to: 11-Solution-Sales Analysis, TableauWorkbooks

Why: Bidirectional filtering increases ambiguity, can create unexpected totals, and can degrade performance.
